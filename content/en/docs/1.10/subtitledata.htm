Subtitle data

    

A subtitle table represents a single line in an ASS file. Apart from regular dialog-lines, it can also be a dialog-style comment line, a "semicolon comment" and a blank line.

 

A subtitle table always contains the following field:

 

kind (string)

One of "dialog", "comment", "scomment" and "blank". Describes what kind of line this is in the subtitle file. You should always test against the value of the kind field before trying to access any other field in a subtitle table.

 

If the value of kind is "blank", no further fields are defined. (It's just a blank line.)

 

If the value is "scomment", the line is a semicolon comment line, that is, a line where the first character is a semicolon. While comments like this is usually ignored during regular processing, you might find them useful to store additional parameters needed for an effect. For "scomment" lines, the following additional field is defined:

 

text (string)

The text following the semicolon, until end of line. This includes any leading and trailing whitespace but never the semicolon or the line break character(s).

 

If the value is "dialog" or "comment", the line has the format of a regular dialog line. The difference is that a "comment" line start with the "Comment:" keyword instead of the "dialog:" keyword, and that comment lines aren't rendered when the subtitles are played back. dialog-style lines contain the following additional fields:

 

layer (number)

The logical layer the line is rendered on. (Lines with a higher layer number are visually rendered on top of lines with a lower layer number.)

 

start_time (number)

end_time (number)

Start and end time of the line, in centiseconds. To get the line durations in milliseconds (used for eg. \t effects), use the following calculation: (end_time - start_time) * 10

 

style (string)

Name of the style associated with this line. Can be used to look up the style in the styles list passed to the process_lines function.

 

name (string)

Name of the character speaking this line. Not used during rendering, but might be useful for storing extra data.

 

margin_l (number)

margin_r (number)

margin_v (number)

Left, right and vertical margin overrides for this line. Margin overriding for an edge is disabled, if the field is 0. Measured in pixels.

 

effect (string)

Special effect to apply to the line. TextSub doesn't complain if this contains an invalid value, so apart from storing actual, valid effects, you can also store additional data here.

 

text (string)

The actual, unprocessed text of the line, including all override and karaoke tags.

 

text_stripped (string)

The text of the line, with all override tags and drawings stripped.

 

karaoke (table)

A list of karaoke syllables in this line. Always contains at least one element. The elements are karaoke tables.

The index n in this table contains the number of syllables.

The syllables are indexed 0 to n-1.

The first syllable in this list is everything before the first karaoke tag in the line, and can often safely be skipped.
